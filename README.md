# Exam Prep Master - 智慧考試準備系統

歡迎使用 **Exam Prep Master**。這是一個專為考生打造的現代化刷題與學習系統，採用 React 與 Tailwind CSS 構建。本系統特別針對「教檢」、「高普考」或各類學科測驗設計，結合了傳統的刷題功能與現代化的 AI 分析輔助，幫助您更有效率地掌握知識盲點。

## 核心功能特色

*   **雙重測驗模式**：
    *   **練習模式 (Practice Mode)**：答題後立即顯示正誤與詳解，並可隨時撰寫筆記與推理過程，適合初次學習。
    *   **考試模式 (Exam Mode)**：模擬真實考場情境，隱藏即時回饋與筆記欄位，直到交卷後才顯示成績，適合考前衝刺。
*   **彈性題數設定**：提供預設題數（10, 20, 50）或「自訂題數」功能，您可以依據當下的時間充裕程度決定練習量。
*   **隨機洗牌 (Shuffle)**：支援題目順序與選項順序的隨機打亂，避免「背答案」的無效學習。
*   **AI 弱點分析匯出**：獨家的 JSON 匯出功能，將您的答題紀錄與錯誤原因格式化，讓您可以直接上傳給 LLM (如 ChatGPT, Claude, Gemini) 進行深度診斷。
*   **響應式設計 (RWD)**：無論使用電腦、平板或手機，都能擁有流暢的操作體驗。

## 使用指南

### 1. 初始設定 (Start Screen)

進入系統後，您會看到設定畫面：

*   **選擇題數 (Number of Questions)**：
    *   點選預設按鈕 (10, 20, 50, All)。
    *   或在輸入框中鍵入您想要的具體數字 (Custom #)。
*   **選擇模式 (Mode)**：
    *   **Practice Mode**：練習用，答錯會立刻亮紅燈，並顯示正確答案。此模式下您可以輸入「推理筆記 (Reasoning)」。
    *   **Exam Mode**：測驗用，介面簡潔，專注於答題速度與準確度。
*   **隨機排列 (Shuffle)**：建議保持開啟 (On)，確保每次練習的題目與選項順序都不同。
*   設定完成後，點擊 **Start Exam** 開始測驗。

### 2. 進行測驗 (Quiz Interface)

*   **作答**：點擊選項即可作答。
*   **導覽**：使用下方的 `Previous` 與 `Next` 按鈕切換題目。
*   **進度條**：上方顯示目前的答題進度與模式。
*   **筆記功能 (僅限練習模式)**：在 "My Reasoning / Notes" 欄位中，您可以打字記錄下為什麼選擇該選項，或者記錄下您對題目的理解。這些文字會被保存並包含在最後的分析報告中。
*   **交卷**：在最後一題，點擊 `Finish Exam`。為防止誤觸，按鈕會變色並顯示 `Confirm Submit`，再次點擊即可交卷。

### 3. 成績與檢討 (Results & Analysis)

交卷後，系統會顯示您的成績單：

*   **Score Card**：顯示準確率 (%) 與得分。
*   **Review Mistakes**：系統會優先列出您答錯的題目。
    *   紅色標示為您的錯誤選擇。
    *   綠色標示為正確答案。
*   **AI 賦能學習 (重要功能)**：
    1.  點擊 **Download for AI Analysis** 按鈕。
    2.  系統會下載一個 `.json` 檔案。
    3.  將此檔案內容複製，或直接上傳給 AI 聊天機器人 (如 ChatGPT)。
    4.  AI 將根據檔案內的 `weakness_analysis_request` 指令，分析您的「錯誤題目」與「推理筆記」，指出您的觀念誤區並提供修正建議。

## 如何新增題目 (自定義題庫)

本系統的題庫來源為 `public/questions-db.json` 檔案。您可以自行編輯此 JSON 檔案來擴充題庫。

### 資料格式規範

請確保檔案內容為一個 JSON Array，格式如下：

```json
[
  {
    "desc": "題目敘述寫在這裡",
    "options": [
      "選項 A",
      "選項 B",
      "選項 C",
      "選項 D"
    ],
    "correct_ans": "選項 B"
  },
  {
    "desc": "第二題...",
    "options": ["...", "...", "...", "..."],
    "correct_ans": "..."
  }
]
```

**注意事項**：
1.  `correct_ans` 的文字必須與 `options` 陣列中的某個選項**完全一致**（包含標點符號）。
2.  系統會自動處理前後空白 (Trim)，但建議保持資料乾淨。

## 技術堆疊 (Tech Stack)

*   **Frontend Framework**: React 19 (Functional Components, Hooks)
*   **Styling**: Tailwind CSS (Utility-first CSS framework)
*   **Icons**: Lucide React
*   **Build Tool**: 無需複雜設定，直接透過 ES Modules 運作於瀏覽器環境（如需本地開發，建議使用 Vite）。
